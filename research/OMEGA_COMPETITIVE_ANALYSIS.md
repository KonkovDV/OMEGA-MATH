# OMEGA: Конкурентный анализ и Позиционирование

Проект **OMEGA** занимает узкую инфраструктурную нишу. В отличие от большинства существующих ИИ-моделей, OMEGA выступает не как "мозг", а как **научно-исследовательская лаборатория (протокол)**.

## 1. Сравнение с флагманскими ИИ-решателями математики

**Конкуренты:** 
*   **AlphaProof и AlphaGeometry 2 (Google DeepMind)**: Закрытые SOTA-модели (LLM + Lean 4 + RL).
*   **FunSearch (DeepMind)**: Поиск программ на основе LLM для решения эволюционных задач.
*   **Numina / AI-MO / Numina-Lean-Agent**: Открытые среды, сфокусированные на олимпиадной математике (Putnam, IMO).
*   **LeanDojo / LLEMMA / DeepSeek-Prover-V2**: Специализированные открытые модели для автодоказательства теорем.

**В чём отличие OMEGA:**
*   **Каталог против Разовых задач:** AlphaProof решает задачи, которые выдает оператор. OMEGA владеет машинно-читаемым каталогом нерешённых проблем и сама выбирает цель через Триаж (T1–T5).
*   **Агностичность к моделям:** OMEGA не конкурирует с DeepSeek или AlphaProof, а может использовать их в качестве "движка разума" для своих агентов.
*   **Прагматизм Триажа:** Часть задач (Т1-класс) OMEGA решает грубой силой (SAT/SMT, SageMath), используя ИИ только программистом-экспериментатором.

## 2. Сравнение с мультиагентными научными фреймворками

**Конкуренты:**
*   **Denario (AstroPilot-AI)**: Фреймворк для написания научных статей с пайплайном (*idea → novelty → methods → results → paper*).
*   **CMBAgent**: Фреймворк с архитектурой Planner-Executor (исторически для космологии).
*   **open-researcher**: Инструменты литературного поиска и маршрутизации контекста.

**В чём отличие OMEGA:**
*   **Строгость к галлюцинациям (Rigorous Evidence):** Denario часто "верит" LLM, что позволяет сгенерировать текст, но не даёт математической точности. В OMEGA есть агент `Prover`, где любые гипотезы должны проходить сертификацию в системе формальных доказательств (Lean 4). 
*   **Полная специализация на математике:** Использование пакета "обязательств доказательства" (`proof_obligations`) вместо обычного описания гипотезы; литературный поиск сцелен на избежание "коллизии патентной новизны".
*   **Роль "Reviewer" с правом вето:** Применение подхода `generate -> referee -> repair`. Агент-критик ищет концептуальные дыры в логике перед публикацией результата.

## Итоговое резюме

| Характеристика | OMEGA | AlphaProof / AI Flagships | Denario / CMBAgent |
| :--- | :--- | :--- | :--- |
| **Главная суть** | Сборочный конвейер (протокол) + база проблем. | Очень умный калькулятор / мозг. | Фреймворк для написания статей / обзоров. |
| **Целевая аудитория** | Исследователи, желающие системно "штурмовать" науки. | Корпорации, Олимпийский комитет IMO. | Учёные, ищущие помощь в написании текста. |
| **Подход к задаче** | Системный каталог $\rightarrow$ Триаж $\rightarrow$ Оркестровка. | Решает то, что дали на вход. | Берёт идею $\rightarrow$ Пишет статью. |
| **Критерий успеха** | Формальное доказательство (Lean 4) / Лог. | Верный ответ. | Оформленный PDF. |
| **Закрытость** | 100% Open Source. | Закрытые веса (обычно). | Open Source. |
