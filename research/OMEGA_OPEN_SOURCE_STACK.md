# OMEGA: Open Source Архитектура и Технологический Стек

Для обеспечения максимальной эффективности, воспроизводимости результатов и сохранения открытой парадигмы проекта, рекомендуется использовать следующий технологический Open Source стек для различных модулей OMEGA. Это позволит избежать "вендор-лока" (проприетарных черных ящиков) на ключевом уровне формальной проверки.

## 1. Вычислительное ядро и Алгоритмы (Experimentalist Module)
Агент-экспериментатор генерирует и исполняет код. Вычислительная база:
*   **SageMath:** Ультимативный математический комбайн (включает GAP, PARI/GP, Singular). Позволяет агенту использовать абстракции высокого уровня.
*   **SAT / SMT-решатели:** Для задач T1 (комбинаторика, теория графов), требующих формализации через логические констрейнты и поиска контрпримеров.
    *   *Z3 (Microsoft Research)* — SMT решатель.
    *   *CaDiCaL / Kissat* — сверхбыстрые SAT солверы.

## 2. Базы данных и Знаний (Librarian & Registry)
*   **PostgreSQL + pgvector:** Реляционное хранение реестра (Registry) и векторный поиск (Vector Store) для RAG (поиск похожих исторических решений и семантический поиск по arXiv).
*   **KùzuDB / Neo4j:** Графовая база данных для построения *Citation Graph* (чтобы исключить циклические зависимости) и *Theorem Dependency Graph*.
*   **OpenAlex (API):** Использование полностью открытого графа научных знаний вместо проприетарных систем для получения достоверной литературы.

## 3. Оркестровка и State Machine (Planner)
*   **LangGraph:** Графовое управление состояниями мультиагентной системы для циклических процессов (`Generate Proof -> Compile -> Repair`).
*   **Temporal / Prefect:** Отказоустойчивые системы оркестровки (Durable execution) для тяжеловесных процессов длительностью несколько часов / суток (особенно для скриптов в классе T1).

## 4. Верификация и Среда выполнения (Prover & Safety)
*   **Lean 4 + Mathlib4:** Золотой стандарт формальных доказательств. `Mathlib4` предоставляет агенту библиотеку уже доказанных лемм.
*   **Pantograph / Lean REPL:** Интерактивные интерфейсы, позволяющие ИИ общаться с Lean шаг за шагом (передача тактик и получение Goal state).
*   **gVisor / E2B Sandbox:** Изолированные firecracker-песочницы, уничтожаемые после каждого запуска, для полностью безопасного исполнения LLM-сгенерированного Python-кода.
*   **Nix:** Менеджер пакетов для строгой фиксации зависимостей в `flake.nix` (стопроцентная воспроизводимость научных результатов).

## 5. ИИ-Движки и Модели (Open Weights)
Локальный или приватный хостинг LLM-агентов.
*   **vLLM:** Сверхбыстрый инференс (PagedAttention) для локального развертывания.
*   **DeepSeek-Prover-V2 / Qwen-2.5-Math:** SOTA модели с открытыми весами, специализирующиеся на математических рассуждениях (Reasoning) и кодогенерации доказательств.
*   **Llama-3 (Instruct):** Подходит для контроля логики (Planner) и критического анализа текста (Reviewer).
