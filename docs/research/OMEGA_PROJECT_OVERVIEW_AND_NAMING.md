# OMEGA: Обзор проекта и Стратегия именования

## Детальное описание проекта

Проект **OMEGA** (Open Mathematics Exploration by Generative Agents) — это амбициозная попытка автоматизировать и структурировать процесс научно-математических открытий с помощью искусственного интеллекта. 

Вместо того чтобы полагаться на "слепую удачу" искусственного интеллекта в виде прямого запроса к абстрактным моделям (напр. ChatGPT), OMEGA создает **строгий научный завод**. Все мировые нерешённые математические проблемы собираются в единый машинно-читаемый каталог. Затем ИИ оценивает их по критерию "вычислительной доступности" (AI-amenability). После этого ИИ-агенты распределяют роли: один ищет работы в литературе (Librarian), второй пишет код для перебора (Experimentalist), третий пытается формально доказать гипотезу в математической среде (Prover в Lean 4), а четвёртый проверяет всё это на ошибки "взглядом критика" (Reviewer).

На академическом уровне OMEGA представляет собой мультиагентную исследовательскую архитектуру. В основе системы лежат три фундаментальных принципа:
1. **Каталогизация и Триаж (Registry & Triage)**: Работа с формализованной базой данных нерешённых проблем и ранжирование от прямого компьютерного перебора (T1) до глубоких логических парадоксов (T5).
2. **Внедрение ролевой мультиагентности**: Оркестрация распределена между изолированными агентами (*Planner, Librarian, Analyst, Experimentalist, Prover, Writer, Reviewer*), что снижает вероятность смысловых галлюцинаций.
3. **Строгая доказательная база (Rigorous Evidence)**: Заявление считается научным только тогда, когда оно подкреплено машиночитаемым журналом вычислений (ledger) или верифицировано в системе формальных доказательств (Lean 4).

## Стратегия именования репозитория и бренда

Использование акронима **OMEGA** остается приоритетным брендом для научных публикаций, презентаций и общего названия архитектуры. Однако для URL-slug (имени репозитория на GitHub) слово `omega` слишком общее и приведет к потере видимости (Name Collision).

Рекомендуемые названия для GitHub:
*   `open-math-agents` (оптимальный баланс узнаваемости)
*   `omega-math`
*   `project-omega`
*   `omega-protocol`

### Проблема с неймингом "OpenMath"
Хотя название **OpenMath** звучит лаконично, оно вступает в прямой конфликт (Academic Name Collision) с уже существующим известным стандартом семантической разметки математики (openmath.org, связанным с MathML). Использование `OpenMath` в чистом виде приведет к путанице среди академического сообщества. Допустимы только комбинированные варианты: `openmath-agents`, `openmath-lab`, `omega-openmath`.

### Английское описание (About Pith)
> OMEGA stands for Open Mathematics Exploration by Generative Agents. It is a standalone architectural protocol that uses AI to systematically map, triage, and solve open math problems through empirical computation and formal Lean 4 verification.
