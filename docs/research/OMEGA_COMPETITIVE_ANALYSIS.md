# OMEGA: конкурентный анализ и позиционирование

Проект **OMEGA** занимает узкую инфраструктурную нишу. В отличие от большинства существующих ИИ-моделей, OMEGA выступает не как «мозг», а как **исследовательский протокол и рабочая среда**.

## 1. Сравнение с флагманскими ИИ-решателями математики

**Конкуренты:** 
*   **AlphaProof и AlphaGeometry 2 (Google DeepMind)**: Закрытые SOTA-модели (LLM + Lean 4 + RL).
*   **FunSearch (DeepMind)**: Поиск программ на основе LLM для решения эволюционных задач.
*   **Numina / AI-MO / Numina-Lean-Agent**: Открытые среды, сфокусированные на олимпиадной математике (Putnam, IMO).
*   **LeanDojo / LLEMMA / DeepSeek-Prover-V2**: Специализированные открытые модели для автодоказательства теорем.

**В чём отличие OMEGA:**
*   **Каталог против разовых задач:** AlphaProof решает задачи, которые выдаёт оператор. OMEGA опирается на машиночитаемый каталог нерешённых проблем и выбирает цель через triage (T1–T5).
*   **Агностичность к моделям:** OMEGA не конкурирует с DeepSeek или AlphaProof, а может использовать их как вычислительный или рассуждающий движок для отдельных агентов.
*   **Прагматизм triage:** часть задач класса T1 OMEGA решает грубой силой (SAT/SMT, SageMath), используя ИИ главным образом как исследовательский инструмент.

## 2. Сравнение с мультиагентными научными фреймворками

**Конкуренты:**
*   **Denario (AstroPilot-AI)**: Фреймворк для написания научных статей с пайплайном (*idea → novelty → methods → results → paper*).
*   **CMBAgent**: Фреймворк с архитектурой Planner-Executor (исторически для космологии).
*   **open-researcher**: Инструменты литературного поиска и маршрутизации контекста.

**В чём отличие OMEGA:**
*   **Строгость к ошибкам рассуждения:** Denario часто полагается на текстовое качество ответа LLM, что полезно для черновиков, но не даёт математической точности. В OMEGA есть агент `Prover`, где гипотезы должны проходить через систему формальных доказательств (Lean 4).
*   **Полная специализация на математике:** используется пакет "обязательств доказательства" (`proof_obligations`) вместо обычного описания гипотезы; литературный поиск нацелен на снижение риска коллизии с уже известными результатами.
*   **Роль `Reviewer` с правом вето:** применяется подход `generate -> referee -> repair`. Агент-критик ищет концептуальные дыры в логике перед публикацией результата.

## Итоговое резюме

| Характеристика | OMEGA | AlphaProof / AI Flagships | Denario / CMBAgent |
| :--- | :--- | :--- | :--- |
| **Главная суть** | Сборочный конвейер и база проблем. | Очень сильный решатель отдельных задач. | Фреймворк для написания статей и обзоров. |
| **Целевая аудитория** | Исследователи, желающие системно работать с открытыми задачами. | Корпорации и исследовательские команды, которым нужен сильный решатель. | Учёные, ищущие помощь в написании текста. |
| **Подход к задаче** | Системный каталог $\rightarrow$ triage $\rightarrow$ оркестровка. | Решает то, что дали на вход. | Берёт идею $\rightarrow$ пишет статью. |
| **Критерий успеха** | Формальное доказательство (Lean 4) / Лог. | Верный ответ. | Оформленный PDF. |
| **Закрытость** | 100% Open Source. | Закрытые веса (обычно). | Open Source. |
